## What It Does

`ats_flutter` adds a single tracing function тАФ `ATS.trace()` тАФ to your Flutter app. Unlike `print()` or `debugPrint()`, ATS traces are:

- **Switchable** тАФ Turn flows on/off without touching code. No more commenting out logs.
- **Structured** тАФ Sequence numbers + call depth let AI reconstruct exact call chains.
- **Persistent** тАФ Knowledge accumulates in `flow_graph.json`, committed to git. AI never starts from scratch.
- **Zero-cost** тАФ In release builds, `ATS.trace()` short-circuits before any logic. Zero overhead.

```yaml

dependencies:
  ats_flutter: ^0.1.0
```


The `init` command creates:
- `.ats/flow_graph.json` тАФ Your knowledge graph
- `lib/generated/ats/ats_generated.g.dart` тАФ Compiled map for O(1) lookups

## Usage

### 1. Initialize at startup

```dart
import 'package:ats_flutter/ats_flutter.dart';
import 'generated/ats/ats_generated.g.dart';

void main() {
  WidgetsFlutterBinding.ensureInitialized();
  AtsGenerated.init(); // O(1) тАФ no JSON parsing, no async, no await
  runApp(MyApp());
}
```

### 2. Add traces to your methods

```dart
class PaymentService {
  Future<void> processPayment(Order order) async {
    ATS.trace('PaymentService', 'processPayment', data: order.toJson());
    // Your business logic stays exactly the same.
    final result = await _gateway.charge(order.total);
    // ...
  }

  Future<void> refund(String transactionId) async {
    ATS.trace('PaymentService', 'refund', data: {'id': transactionId});
    await _gateway.refund(transactionId);
  }
}
```

### 3. Define flows in flow_graph.json

```json
{
  "ats_version": "4.0.0",
  "project": "my_app",
  "flows": {
    "PAYMENT_FLOW": {
      "description": "Checkout and payment processing",
      "active": false,
      "classes": {
        "PaymentService": {
          "methods": ["processPayment", "refund"],
          "last_verified": "2026-04-16"
        }
      }
    }
  }
}
```

### 4. Activate when debugging

```bash
dart run ats_flutter activate PAYMENT_FLOW
# тЖТ Flow activated. Hot Restart (Shift+R) to see logs.
```

Console output:
```
[ATS][PAYMENT_FLOW][#001][d0] CheckoutBloc.onCheckout | {"cart_id": "abc"}
[ATS][PAYMENT_FLOW][#002][d1] PaymentService.processPayment | {"amount": 99}
[ATS][PAYMENT_FLOW][#003][d2] StripeGateway.charge | {"status": "ok"}
```

### 5. Silence when done

```bash
dart run ats_flutter silence PAYMENT_FLOW
# тЖТ Code stays clean. No logs to remove. Ever.
```

## V4 Log Format

```
[ATS][FLOW_NAME][#SEQ][dDEPTH] Class.method | {optional_data}
```

| Token | Purpose | Example |
|---|---|---|
| `FLOW_NAME` | Which business flow is being traced | `PAYMENT_FLOW` |
| `#SEQ` | Global execution order тАФ AI reads which ran first | `#003` |
| `dDEPTH` | Call stack depth тАФ AI infers who called whom | `d2` (2 levels deep) |
| `Class.method` | Exact source code location | `StripeGateway.charge` |
| `{data}` | Optional JSON payload for debugging | `{"status": "ok"}` |

**Why this matters:** Sequence + Depth together let AI reconstruct the full call tree from flat console output. No breakpoints, no step-debugging, no source-level tracing needed.

## How It Works Under the Hood

### CodeGen Architecture

ATS does **not** bundle JSON into your APK. Instead:

1. `ats sync` compiles `flow_graph.json` тЖТ `ats_generated.g.dart` (pure `const Map`)
2. `AtsGenerated.init()` loads the map at startup тАФ O(1), synchronous
3. `ATS.trace()` does a map lookup:
   - Method belongs to active flow тЖТ print structured log
   - Method not active тЖТ **return immediately** (no-op)
4. Hot Restart reloads the generated file тЖТ changes take effect instantly

```dart
// AUTO-GENERATED BY ATS CLI тАФ DO NOT EDIT
import 'package:ats_flutter/ats_flutter.dart';

const _kMethodMap = <String, List<String>>{
  'PaymentService.processPayment': ['PAYMENT_FLOW'],
  'PaymentService.refund': ['PAYMENT_FLOW'],
};

const _kActiveFlows = <String>['PAYMENT_FLOW'];

abstract class AtsGenerated {
  static void init() {
    ATS.internalInit(_kMethodMap, _kActiveFlows);
  }
}
```

### Zero Production Overhead

```dart
static void trace(String className, String method, {dynamic data}) {
  if (kReleaseMode) return;  // тЖР First line. Zero cost in production.
  // ... rest only runs in debug/profile mode
}
```
