# Changelog

## 0.2.0 — V5 Architecture (Runtime-Only)

### Breaking Changes
- **Removed CLI** — `ats init`, `ats sync`, `ats activate`, `ats silence`, `ats dashboard` commands are no longer part of this package. Use the MCP Server instead.
- **Removed `executables`** from pubspec.yaml — `dart pub global activate ats_flutter` no longer installs a CLI.
- **Removed dependencies** — `args`, `path`, `yaml` packages removed (were only needed by CLI).

### Why
The MCP Server (TypeScript) now handles all CLI operations, code generation, and AI tooling universally across all languages. This package is now a pure **Runtime SDK** — it only provides `ATS.trace()` and supporting classes that run inside your Flutter app.

### Migration
- Replace `dart run ats_flutter init` → MCP Server handles this
- Replace `dart run ats_flutter sync` → MCP auto-syncs on `ats_activate`/`ats_silence`
- Replace `dart run ats_flutter activate FLOW` → AI calls `ats_activate` via MCP or use Web Dashboard
- Replace `ats dashboard` → `npx ats-mcp-server dashboard` or via MCP auto-start

### What's Unchanged
- `ATS.trace()` — same API, same behavior
- `ATS.internalInit()` — same API
- `ATS.resetSequence()` — same API
- `FlowRegistry` — same O(1) lookup
- `LogWriter` — same file logging
- Generated `ats_generated.g.dart` — same format, now generated by MCP Server (TypeScript) instead of Dart CLI

## 0.1.0 — Initial Release

- `ATS.trace()` with sequence tracking and depth inference
- `FlowRegistry` — O(1) method→flow lookup
- `LogWriter` — JSONL file logging
- CLI: init, sync, activate, silence, status, graph, dashboard
- V4 log format: `[ATS][FLOW][#SEQ][dDEPTH] Class.method | {data}`
- Muted methods support (V4)
- Zero production overhead (`kReleaseMode` guard)
